## 🔧 Требования

- Node.js 18+ 
- npm или yarn
- Expo CLI
- Аккаунт Supabase (бесплатный)

## 📦 Установка зависимостей

```bash
npm install --legacy-peer-deps
```

## 🗄️ Настройка Supabase

### 1. Создайте проект в Supabase

1. Перейдите на [supabase.com](https://supabase.com)
2. Создайте новый проект
3. Дождитесь завершения настройки (2-3 минуты)

### 2. Выполните SQL скрипт

1. Откройте SQL Editor в Supabase Dashboard
2. Скопируйте содержимое файла `DATABASE_SCHEMA.sql`
3. Выполните скрипт

Это создаст:
- Таблицы: `profiles`, `workouts`, `user_stats`, `achievement_templates`, `user_achievements`
- Триггеры для автоматического обновления статистики
- Row Level Security политики
- Начальные достижения

### 3. Получите API ключи

1. Перейдите в Settings -> API
2. Скопируйте:
   - Project URL
   - anon/public key

## 🔐 Настройка переменных окружения

1. Создайте файл `.env.local`:

```bash
cp .env.local.example .env.local
```

2. Заполните переменные:

```env
EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
```

## 🚀 Запуск приложения

### Для разработки:

```bash
# Запуск Metro Bundler
npx expo start

# Для iOS
npx expo start --ios

# Для Android
npx expo start --android

# Для Web
npx expo start --web
```

### Для тестирования на устройстве:

1. Установите Expo Go на телефон
2. Отсканируйте QR-код из терминала

## 📊 База данных

### Таблицы:

1. **profiles** - Профили пользователей
   - Расширяет auth.users
   - Уровень, XP, аватар

2. **workouts** - Тренировки
   - Дистанция, время, темп
   - GPS маршрут (JSONB)
   - Калории

3. **user_stats** - Статистика (кэш)
   - Общая дистанция, время
   - Лучшие результаты
   - Серии тренировок

4. **achievement_templates** - Шаблоны достижений
   - Название, описание, иконка
   - Требования, награда XP

5. **user_achievements** - Полученные достижения
   - Связь пользователь-достижение
   - Дата получения

### Автоматизация:

- **Триггер** обновляет статистику после каждой тренировки
- **RLS** защищает данные пользователей
- **Функция** создаёт профиль при регистрации

## 🔒 Безопасность

- Row Level Security (RLS) на всех таблицах
- Пользователи видят только свои данные
- Аутентификация через Supabase Auth
- Токены хранятся в AsyncStorage

## 🐛 Решение проблем

### Ошибка "Cannot find module 'react-native-worklets'"

```bash
npm install react-native-worklets@0.5.1 --legacy-peer-deps
```

### Карты не работают в Expo Go

Это нормально. Для карт нужен Development Build:

```bash
eas build --profile development --platform ios
```

### Ошибка подключения к Supabase

Проверьте:
1. Правильность URL и ключа в `.env.local`
2. Выполнен ли SQL скрипт
3. Включен ли RLS на таблицах
